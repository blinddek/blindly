## Objective

Create the saved quotes, swatch requests, and measure requests schema. These are lead capture mechanisms that feed the sales pipeline — customers who aren't ready to buy immediately but have shown intent.

---

## Context

Three lead capture flows in Blindly:

1. **Saved Quotes** — customer configures blinds, saves their cart as a quote with a shareable link. Follow-up emails sent at 24h, 72h, and 7d if not converted.
2. **Swatch Requests** — during colour selection (Step 5), customer can request a free physical swatch sample mailed to them.
3. **Measure Requests** — at the measurement step (Step 6), customer can request a professional in-home or virtual measurement session.

All three are visible in the admin dashboard as lead pipeline items.

## Key Schema Details

### Saved Quotes
- `quote_token` — unique random string used in shareable URL: `/quote/abc123xyz`
- `cart_data` — full JSONB snapshot of the cart at save time (all blinds with config, prices, extras)
- `converted_to_order_id` — FK to orders table, set when the quote becomes an actual order
- `expires_at` — defaults to 30 days from creation
- Follow-up flags (`email_sent_24h`, `email_sent_72h`, `email_sent_7d`) — used by cron/automation to track which reminder emails have been sent

### Swatch Requests
Status pipeline: `new → sent → delivered`
- Captures the exact blind range and colour the customer wants to sample
- Delivery address for posting the physical swatch

### Measure Requests
Status pipeline: `new → contacted → scheduled → completed → cancelled`
- `preferred_method`: 'in_person' or 'virtual' (video call measurement guidance)
- `scheduled_at`: when the measurement is booked for

## Notes for Claude Code

- The exact SQL is in BUILD_INDEX.md under "Migration 005 — Quotes, Swatch Requests & Lead Capture"
- The `quote_token` should be generated in application code using `nanoid` (installed in Build 01) — not auto-generated by the database
- `cart_data` stores the entire cart as-is at the moment of saving — this is intentional. If prices change later, the quote still reflects what the customer saw.
- Swatch and measure requests don't require user authentication — they're anonymous lead capture forms
